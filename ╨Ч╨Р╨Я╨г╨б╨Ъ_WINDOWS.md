# Запуск на Windows (GPU 8 ГБ)

Пошагово. Все команды — в **PowerShell**, из папки проекта.

---

## 0. Что должно быть на машине

- **NVIDIA GPU** (8 ГБ VRAM достаточно) и свежий драйвер.
- **Python 3.10 или 3.11** (не 3.12+ — под неё не все колёса собраны).
  Проверка: `py -3.11 --version`
- **Git for Windows** (в нём уже есть `git-lfs`).
- **~40 ГБ свободного места**: сама модель Qwen3-4B весит ~8 ГБ, плюс кэш и адаптеры.

Проверь, что видеокарта видна:

```powershell
nvidia-smi
```

В выводе должна быть твоя карта и версия CUDA. Если команды нет — поставь драйвер NVIDIA.

---

## 1. Виртуальное окружение и зависимости

```powershell
cd путь\до\qwen3-alignment-explainer

py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip

# torch с CUDA — ставится ПЕРВЫМ и отдельно, с индекса PyTorch
pip install torch==2.6.0 --index-url https://download.pytorch.org/whl/cu124

# всё остальное
pip install -r requirements.txt
```

> Если `Activate.ps1` не запускается («выполнение сценариев отключено»), один раз выполни:
> `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`

Проверка, что CUDA реально работает (**обязательно, до обучения**):

```powershell
python -c "import torch; print(torch.cuda.is_available(), torch.cuda.get_device_name(0))"
```

Должно напечатать `True` и название карты. Если `False` — torch встал в CPU-версии,
переустанови его командой из шага выше.

---

## 2. Запуск ноутбука

```powershell
jupyter notebook
```

Открой `notebooks/alignment_pipeline.ipynb` и запусти **Cell → Run All** (сверху вниз, без правок).

Первый запуск скачает модель Qwen3-4B (~8 ГБ) с Hugging Face — нужен интернет, токен не требуется.

### Сколько это займёт

На карте уровня RTX 3070 (8 ГБ) — **примерно 3–4 часа суммарно**:

| Стадия | Время |
|---|---|
| скачивание модели | 10–20 мин (один раз) |
| базовый замер + SFT (задача 1) | ~45 мин |
| DPO по стилю (задача 2) | ~40 мин |
| Reward Model (задача 3) | ~40 мин |
| DPO по качеству (задача 4) | ~50 мин |
| SimPO (задача 5) | ~40 мин |

Ноутбук печатает время и пиковую VRAM после каждой стадии.

**Не закрывай ноутбук между стадиями** — но если сессия всё-таки упала, ничего не потеряно:
каждая стадия сохраняет свой адаптер в `adapters/`, и следующая грузит его с диска.

---

## 3. Если не хватило видеопамяти (`CUDA out of memory`)

Ноутбук настроен под 8 ГБ с запасом, но если карта занята другими программами
(браузер, игры, второй монитор) — закрой их и перезапусти ядро. Если ошибка осталась,
в ячейке №1 уменьши:

```python
MAX_LEN = 384          # было 448
MAX_NEW_TOKENS = 192   # было 256
```

и в конфиге упавшей стадии поставь `per_device_train_batch_size=1`,
увеличив `gradient_accumulation_steps` вдвое (чтобы эффективный batch не изменился).

---

## 4. Что должно получиться

В конце ноутбук печатает таблицу из пяти метрик и сохраняет `results/final_metrics.json`.
Оттуда берутся буквы ответов для заданий 1–5 на платформе.

Также появятся:
- `adapters/` — обученные адаптеры (sft, dpo_style, rm, dpo_quality, simpo);
- `results/generations_*.json` — ответы модели на public_test_style с их `P_simple`;
- `results/final_metrics.json` — итоговые числа.
