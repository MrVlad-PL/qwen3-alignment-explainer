# Объяснятель: выравнивание языковой модели

Полный цикл alignment компактной модели `Qwen/Qwen3-4B-Instruct-2507` под задачу «объясняй просто,
но не теряй смысл»: **SFT → DPO(стиль) → Reward Model → DPO(качество) → SimPO**.

Всё обучение — QLoRA (4-bit NF4) внутри одного ноутбука:
[`notebooks/alignment_pipeline.ipynb`](notebooks/alignment_pipeline.ipynb).
Ноутбук запускается сверху вниз без ручных правок, `seed = 42`, генерация greedy (`do_sample=False`).

## Как устроен пайплайн

```
base ──SFT(kid_adult)──> [задача 1] ──DPO(kid_adult)──> [задача 2] ─┬──DPO(good_bad)──> [задача 4]
                                                                     └──SimPO(good_bad)──> [задача 5]

base ──Reward Model(good_bad)──> [задача 3]     (обучается отдельно)
```

Задачи 4 и 5 стартуют из **одной и той же** точки — модели после задачи 2. Иначе сравнение
DPO и SimPO было бы нечестным: разные стартовые модели, а не разные алгоритмы.

### Три решения, которые здесь важны

**Reference-модель в DPO — это модель после SFT, а не сырая база.** DPO штрафует уход политики
от reference. Если якорем сделать сырую базу, DPO будет тянуть модель обратно к сложному регистру —
ровно к тому, от чего мы уходили на SFT. Поэтому стартовый адаптер грузится **дважды**: копия
`default` обучается, копия `reference` заморожена и служит якорем (`ref_adapter_name` в TRL).
Бонус: вторая полная модель в VRAM не нужна, что критично на 8 ГБ.

**Нормировка на длину в метриках задач 4–5.** Метрика — средний logprob **на токен** ответа, а не
сумма. Сырая сумма logprob систематически завышает короткие ответы, и метрика перестала бы измерять
качество, начав измерять длину.

**Контроль на поверхностный признак у Reward Model.** В `good_bad` плохие ответы в среднем длиннее
хороших, поэтому тривиальное правило «короче = лучше» само по себе даёт заметную accuracy, ничего не
понимая по сути. Ноутбук печатает этот baseline рядом с метрикой задачи 3: разрыв между ними —
единственное доказательство, что оценщик выучил смысл, а не форму.

## Структура

```
notebooks/alignment_pipeline.ipynb   весь пайплайн: обучение + все пять метрик
data/                                kid_adult, good_bad (обучение)
                                     public_test_style, public_test_quality (только метрики)
metrics/style_clf.pkl                измеритель простоты: TF-IDF (word + char_wb) + LogisticRegression
metrics/gold_rm/                     эталонный судья качества (LoRA + value head)
results/                             метрики и сгенерированные ответы (появляются после прогона)
requirements.txt                     зафиксированные версии
ЗАПУСК_WINDOWS.md                    как запустить на Windows + GPU
COMMIT_PLAN.md                       порядок коммитов по ходу работы
```

## Воспроизведение

Colab (T4): открыть ноутбук, задать `REPO_URL`, Run all.
Локально (Windows, GPU 8 ГБ): см. [ЗАПУСК_WINDOWS.md](ЗАПУСК_WINDOWS.md).

Все пять метрик считаются ячейками ноутбука и печатаются вместе с выбранным интервалом;
итог дублируется в `results/final_metrics.json`.
